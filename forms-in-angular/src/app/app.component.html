Forms In Angular

Angular provides two different approaches to handle user inputs through forms:

  1. Reactive (model driven)
  2. Template driven

Scalability

If forms are a central part of your application, scalability is very important. Being able to use form models across components is critical.

Reactive forms are more scalable than template-driven forms. They provide direct access to underlying Form API, and use synchronous data flow between the view and data model, which makes creating large scale forms easier.

Template-driven forms on the other hand focus on simple scenarios and are not as resuable. They abstract away the underlying Form API, and use asynchronous data flow between the view and data model. The abstraction of template-driven forms also affects testing.

Common form foundation classes

  • FormControl: Tracks the value and validation status of an individual form control.
  • FormGroup: Tracks the same values and status for a collection of form controls.
  • FormArray: Tracks the same values and status for an array of form controls.
  • ControlValueAccessor: Creates a bridge between Angular FormControl instances and built-in DOM elements.

Form Validation

Validation is an integral part of managing any set of forms. Whether you're checking for required fields or querying an external API for an existing username, Angular provides a set of built-in validators as well as the ability to create custom validators.

  • Reactive forms: Define custom validators as functions that receive a control to validate
  • Template-driven forms: Tied to template directives, and must provide custom validator directives that wrap validation functions
